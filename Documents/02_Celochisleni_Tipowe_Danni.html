02 Celochisleni Tipowe Danni

Целочислени типове данни

1. Константи от целочислен тип (цели числа)

Започва със знак + или - или е без знак, след което следва последователност от цифри.

2. Променливи

Всяка променлива, която се използва в програмата, трябва да бъде дефинирана. При дефинирането на променлива се заделя оперативна памет за тази променлива.

Пример:

int a;  // int - тип данни, a - име на променлива (идентификатор), ; - край на оператора (декларация)

Терминология:

  • int - целочислен тип данни (integer), 4 bytes (32 bits) със знак. Може да съхранява положителни и отрицателни цели числа
  • Променлива - именувана област от паметта, която съхранява стойност, която може да се променя по време на изпълнение на програмата
  • Дефиниране на променлива - процесът на заделяне на памет и свързване на име с тази памет

Типови модификатори:

  • по дължина: short, long
    • short - намалява размера на типа (обикновено 2 bytes)
    • long - увеличава размера на типа (обикновено 8 bytes на 64-битови системи)
  • по знак: signed, unsigned
    • signed - тип със знак, може да съхранява положителни и отрицателни числа (по подразбиране)
    • unsigned - тип без знак, може да съхранява само положителни числа (включително 0), което удвоява максималната стойност

Примери:

unsigned int b;              // unsigned - модификатор без знак, int - тип, b - име на променлива, ; - край на оператора
short int c;                 // short - модификатор за дължина, int - тип, c - име на променлива, ; - край на оператора
unsigned short int d;        // unsigned short - модификатори, int - тип, d - име на променлива, ; - край на оператора

Операция sizeof(int) - резултат - брой байтове, които се отделят за дадения тип (в случая 4)

sizeof(a) - ако a е променлива от тип int - резултатът е същият като sizeof(int)

Терминология:

  • sizeof() - операция (не функция), която връща размера в байтове на даден тип или променлива. Полезно за определяне на размера на данни в паметта

3. Операции

  • събиране (+)
  • изваждане (-)
  • умножение (*)
  • деление (/)
  • остатък при целочислено деление (%)

Терминология:

  • % (модуло) - операция за остатък при целочислено деление. Например 15 % 4 дава 3 (остатъкът при деление на 15 на 4)

При целочислено деление - цяло/цяло - резултатът е цяло число

Пример: 15/4 е 3

4. Оператор за присвояване

Синтаксис:

променлива = израз;  // = - оператор за присвояване, ; - край на оператора

5. Въвеждане и извеждане на данни от целочислен тип

#include <stdio.h>          // препроцесорна директива - включва стандартната библиотека за вход/изход (виж [Препроцесор](../01_Въведение%20в%20С.md#4-препроцесор-и-етапи-на-компилация))

int a;                      // int - тип данни (виж [Променливи](#2-променливи)), a - име на променлива, ; - край на оператора
scanf("%d", &a);            // scanf - функция за вход от #include <stdio.h>, "%d" - форматиращ низ, &a - адрес на променливата (оператор &), ; - край на оператора

printf("a=%d", a);          // printf - функция за изход от #include <stdio.h>, "a=%d" - форматиращ низ, a - стойност за извеждане, ; - край на оператора

Ако стойността на a е например 10, на екрана се извежда:

a=10

Терминология:

  • scanf() - функция за форматиран вход (scan formatted), чете данни от стандартния вход (обикновено клавиатурата). Идва от #include <stdio.h> - препроцесорната директива включва стандартната библиотека за вход/изход (виж Препроцесор)
  • printf() - функция за форматиран изход (print formatted), извежда данни на стандартния изход (обикновено екрана). Идва от #include <stdio.h> - препроцесорната директива включва стандартната библиотека за вход/изход (виж Препроцесор)
  • %d - форматна спецификация за цели числа (decimal). Указва на scanf() и printf() как да интерпретират/форматират данните
  • &a - оператор за адрес (&). При scanf() е задължителен, защото функцията трябва да знае къде в паметта да запише прочетената стойност. &a дава адреса на променливата a
