## 1. Целочислени типове данни

> **От конспекта:** Целочислени типове данни - константи, променливи, операции, въвеждане и извеждане на данни от целочислен тип. Представяне на целите числа в паметта. Диапазон от допустими стойности.

### 1.1. Константи от целочислен тип

Целочислена константа започва със знак `+` или `-`, или е без знак, следвана от последователност от цифри.

**Примери:** `245`, `-10`, `+7`, `0`

### 1.2. Променливи

Всяка променлива трябва да бъде **дефинирана**, преди да се използва. При дефинирането се заделя оперативна памет (ОП) за нея.

```c
int a;   // int - целочислен тип данни, a - име на променлива (идентификатор), ; - край на оператора
```

- **Променлива** – именувана област от паметта, чиято стойност може да се променя по време на изпълнение.
- **Дефиниране на променлива** – заделяне на памет и свързване на име с тази памет.

**Типови модификатори:**

| Модификатор | Значение | Типичен размер | Диапазон |
|---|---|---|---|
| `short` | намалява дължината | 2 байта | -32768 до 32767 |
| `long` | увеличава дължината | 4 или 8 байта (зависи от системата) | по-голям |
| `signed` | със знак (по подразбиране) | – | отрицателни и положителни |
| `unsigned` | без знак | – | само положителни (вкл. 0), удвоена горна граница |

```c
unsigned int b;        // без знак
short int c;            // 2 байта, със знак
unsigned short int d;   // 2 байта, без знак
```

**Операцията `sizeof`** връща броя байтове, заделени за тип или променлива: `sizeof(int)`, `sizeof(a)`.

### 1.3. Представяне на целите числа в паметта

Всяко цяло число се пази в паметта като поредица от битове (0 и 1).

- **Числа без знак (`unsigned`)** – директен двоичен код. Пример: `5` в 1 байт е `00000101`.
- **Числа със знак (`signed`)** – използва се **допълнителен код (two's complement)**: положителните числа се пазят в обикновен двоичен вид, а отрицателните се получават чрез инвертиране на битовете на съответното положително число и добавяне на 1.

**Пример (1 байт, за нагледност):**

| Число | Двоичен запис (1 байт) |
|---|---|
| `5` | `00000101` |
| `-5` | `11111011` (инверсия на `00000101` = `11111010`, `+1` = `11111011`) |

Най-старшият бит показва знака при числа със знак (0 = положително, 1 = отрицателно).

### 1.4. Диапазон от допустими стойности

За тип с `n` бита:

- **Със знак:** от \(-2^{n-1}\) до \(2^{n-1}-1\)
- **Без знак:** от `0` до \(2^{n}-1\)

| Тип | Размер | Диапазон (със знак) | Диапазон (без знак) |
|---|---|---|---|
| `char` | 1 байт (8 бита) | -128 до 127 | 0 до 255 |
| `short int` | 2 байта (16 бита) | -32768 до 32767 | 0 до 65535 |
| `int` | 4 байта (32 бита) | -2147483648 до 2147483647 | 0 до 4294967295 |
| `long int` | 4 или 8 байта | зависи от системата | зависи от системата |

**Защо е важно:** ако резултат от изчисление излезе извън диапазона на типа, се получава **препълване (overflow)** и стойността "прескача" в началото на диапазона (без грешка при компилация или изпълнение!).

```c
short s = 32767;
s = s + 1;              // препълване: s става -32768, а не 32768
```

### 1.5. Операции

`+` (събиране), `-` (изваждане), `*` (умножение), `/` (деление), `%` (остатък при целочислено деление).

- При целочислено деление **цяло/цяло** резултатът е цяло число: `15 / 4` е `3`.
- `%` (модуло) връща остатъка: `15 % 4` е `3`.

### 1.6. Оператор за присвояване

```c
променлива = израз;   // = - оператор за присвояване (различен от == за сравнение!)
```

### 1.7. Въвеждане и извеждане на данни от целочислен тип

```c
#include <stdio.h>       // включва библиотеката за вход/изход

int main(void) {
    int a;                    // дефиниране на променлива
    printf("Въведете a: ");   // printf - извеждане, "%d" не е нужен тук, само текст
    scanf("%d", &a);           // scanf - въвеждане, %d - спецификатор за int, &a - адрес на a
    printf("a=%d\n", a);       // извеждане на стойността на a
    return 0;
}
```

Ако въведем `10`, изходът е: `a=10`

- **`scanf`/`printf`** – функции за форматиран вход/изход от `<stdio.h>`.
- **`%d`** – спецификатор за формат за тип `int` (decimal).
- **`&a`** – адрес на променливата `a` (задължителен при `scanf`, за да знае къде да запише стойността).

**Малка задача (от конспекта):** размяна на две целочислени променливи, със и без помощна променлива.

```c
int a = 5, b = 10, temp;
temp = a;   // с помощна променлива
a = b;
b = temp;

// без помощна променлива:
a = a + b;
b = a - b;
a = a - b;
```

## 2. Реални типове данни

> **От конспекта:** Реални типове данни - константи, променливи, допустими операции, въвеждане и извеждане на данни от реални типове.

### 2.1. Константи от реален тип

**а) С десетична точка:** `2.33`, `-12.4`, `0.123`

**б) С експонента (мантиса и порядък):** `2.5E-1` = \(2.5 \times 10^{-1}\) = `0.25`, където `2.5` е мантисата, а `-1` е порядъкът (цяло число).

### 2.2. Променливи от реален тип

```c
float a;     // 4 байта ОП
double b;    // 8 байта ОП
```

Разликата между `float` и `double` е в **диапазона от допустими стойности** и в **точността** (брой значещи цифри). `double` е по-точен и с по-голям диапазон.

### 2.3. Операции и изрази

Допустими операции: събиране, изваждане, умножение, деление.

- Ако **един** от операндите е реален, резултатът е реален.

```c
int s, n;
float avr;

avr = s / n;          // ГРЕШНО: цяло/цяло дава цяло число (загуба на точност)
avr = (1.0 * s) / n;  // ВЯРНО: (1.0*s) е реално число
avr = (float)s / n;   // ВЯРНО: (float) - явно преобразуване на типа (type cast)
```

```c
int n;
float s, avr;

avr = s / n;   // ВЯРНО: реално/цяло дава реално число
```

### 2.4. Въвеждане и извеждане

```c
#include <stdio.h>

int main(void) {
    float a;
    double b;

    scanf("%f", &a);     // %f - спецификатор за float
    scanf("%lf", &b);    // %lf - спецификатор за double при въвеждане

    printf("a=%f  b=%lf\n", a, b);   // при printf за float и double се използва %f
    return 0;
}
```

**Бележка за производителност:** операциите с `double` са леко по-скъпи от тези с `float` (повече байтове за пренасяне и изчисление), но разликата е незначителна на съвременни процесори. Препоръка: използвайте `double` по подразбиране за по-добра точност, освен ако паметта е строго ограничена (напр. големи масиви от милиони елементи), където `float` пести памет наполовина.

## 3. Условен оператор

> **От конспекта:** Условен оператор. Пълна и непълна форма - синтаксис и начин на изпълнение. Вложени условни оператори.

### 3.1. Пълна форма

**Синтаксис:**

```c
if (израз)
    оператор1
else
    оператор2
```

**Начин на изпълнение:** изчислява се `израз`. Ако е **истина** (различно от 0), изпълнява се `оператор1`, иначе се изпълнява `оператор2`.

```c
int a = 5, b = 3;

if (a > b) {                              // if - условен оператор
    printf("%d е по-голямо от %d\n", a, b);
} else {                                  // else - изпълнява се, ако условието е лъжа
    printf("%d не е по-голямо от %d\n", a, b);
}
```

### 3.2. Непълна форма

**Синтаксис:**

```c
if (израз)
    оператор
```

**Начин на изпълнение:** ако `израз` е истина, се изпълнява `оператор`; иначе програмата продължава напред, без да прави нищо.

```c
int n = -5;

if (n < 0) {
    printf("Числото е отрицателно\n");
}
```

### 3.3. Вложени условни оператори

Вътре в тялото на `if` или `else` може да има отново `if` – това се нарича **вложен условен оператор**. Използва се за проверка на повече от две възможности.

```c
int n;
scanf("%d", &n);

if (n > 0) {
    printf("Положително\n");
} else {
    if (n < 0) {                 // вложен if в else-клона
        printf("Отрицателно\n");
    } else {
        printf("Нула\n");
    }
}
```

Тази конструкция обикновено се пише по-компактно с `else if`:

```c
if (n > 0) {
    printf("Положително\n");
} else if (n < 0) {
    printf("Отрицателно\n");
} else {
    printf("Нула\n");
}
```

### 3.4. Съставен оператор (напомняне)

Няколко оператора, обградени с `{ }`, се третират като **един** оператор – затова е задължителен, когато тялото на `if`/`else` съдържа повече от един ред код.

### 3.5. Оператори за сравнение и логически оператори (използват се в условията)

| Оператор | Значение |
|---|---|
| `==` | равно |
| `!=` | различно |
| `>`, `<`, `>=`, `<=` | сравнение |
| `&&` | логическо И |
| `\|\|` | логическо ИЛИ |
| `!` | логическо НЕ |

```c
int age = 20;
if (age >= 18 && age <= 65) {     // && - истина само ако и двете условия са истина
    printf("Възрастта е валидна\n");
}
```

## 4. Оператори за цикъл

> **От конспекта:** Оператори за цикъл - с предусловие и постусловие, оператор за цикъл `for` - синтаксис и начин на изпълнение. Вложени цикли. Оператор за прекратяване на цикъл (`break`).

### 4.1. Цикъл с предусловие (`while`)

**Синтаксис:**

```c
while (израз)
    оператор
```

**Изпълнение:** условието се проверява **преди** тялото. Ако е лъжа от началото, тялото не се изпълнява нито веднъж.

```c
int count = 1;
while (count <= 5) {         // проверка ПРЕДИ тялото
    printf("%d ", count);
    count = count + 1;
}
// Изход: 1 2 3 4 5
```

### 4.2. Цикъл с постусловие (`do-while`)

**Синтаксис:**

```c
do
    оператор
while (израз);
```

**Изпълнение:** тялото се изпълнява **поне веднъж**, след което се проверява условието; ако е истина, цикълът се повтаря.

```c
int n;
do {
    printf("Въведете положително число: ");
    scanf("%d", &n);
} while (n <= 0);        // проверка СЛЕД тялото
```

### 4.3. Цикъл `for`

**Синтаксис:**

```c
for (инициализация; условие; стъпка)
    оператор
```

**Изпълнение:**

1. Изпълнява се **инициализацията** (веднъж, в началото).
2. Проверява се **условието** – ако е истина, изпълнява се тялото.
3. Изпълнява се **стъпката**.
4. Връщане на стъпка 2.

```c
int i;
for (i = 1; i <= 5; i = i + 1) {
    printf("%d ", i);
}
// Изход: 1 2 3 4 5
```

**Кога `for`, кога `while`:**

| | `for` | `while` |
|---|---|---|
| Подходящ, когато | знаем точния брой повторения | не знаем броя повторения предварително |
| Брояч | в заглавието (компактно) | инициализация преди, стъпка в тялото |

Всеки `for` цикъл може да се препише като `while` и обратно – изборът е въпрос на четимост.

### 4.4. Вложени цикли

Цикъл в тялото на друг цикъл. Използва се например за обхождане на двумерни структури (таблици) или комбинации от две променливи.

```c
// Таблица за умножение 1..3
int i, k;
for (i = 1; i <= 3; i = i + 1) {        // външен цикъл - редове
    for (k = 1; k <= 3; k = k + 1) {    // вътрешен цикъл - колони
        printf("%4d", i * k);
    }
    printf("\n");
}
/* Изход:
   1   2   3
   2   4   6
   3   6   9
*/
```

### 4.5. Оператор за прекратяване на цикъл (`break`)

`break` веднага прекратява **най-вътрешния** цикъл (или `switch`), в който се намира, и изпълнението продължава след него.

```c
int n = 17, i, isPrime = 1;

for (i = 2; i < n; i = i + 1) {
    if (n % i == 0) {
        isPrime = 0;
        break;              // намерихме делител - няма смисъл да продължаваме цикъла
    }
}
printf(isPrime ? "Просто число\n" : "Не е просто\n");
```

**Бележка за производителност:** `break` спестява ненужни итерации – в примера по-горе, щом открием делител, спираме веднага, вместо да проверяваме до `n-1`. За проверка на просто число по-ефективен вариант е цикълът да върви само до `sqrt(n)` вместо до `n-1`, защото ако `n` има делител по-голям от `sqrt(n)`, задължително има и по-малък от `sqrt(n)` – това намалява сложността от O(n) на O(√n).

## 5. Символен тип данни. ASCII таблица

> **От конспекта:** Символен тип данни. ASCII таблица. Константи, променливи, операции, въвеждане и извеждане на данни от символен тип.

### 5.1. Тип `char`

`char` съхранява **един символ** и заема **1 байт** памет. Всъщност `char` е малко цяло число – всеки символ се пази чрез своя **ASCII код**.

```c
char c;          // 1 байт
c = 'A';         // символна константа - в единични кавички
```

- **Символна константа** – единичен символ в единични кавички: `'A'`, `'5'`, `'\n'` (нов ред), `'\0'` (нулев байт).
- Диапазон на `char`: -128 до 127 (`signed char`) или 0 до 255 (`unsigned char`), зависи от компилатора.

### 5.2. ASCII таблица

**ASCII (American Standard Code for Information Interchange)** е стандартна таблица, която задава на всеки символ (буква, цифра, знак) числов код от 0 до 127.

| Символ | ASCII код |
|---|---|
| `'0'`–`'9'` | 48–57 |
| `'A'`–`'Z'` | 65–90 |
| `'a'`–`'z'` | 97–122 |
| `' '` (интервал) | 32 |
| `'\n'` (нов ред) | 10 |
| `'\0'` (нулев байт) | 0 |

Тъй като `char` е число, символите могат да се сравняват и с тях могат да се правят аритметични операции:

```c
char c = 'A';
printf("%d\n", c);           // извежда 65 (ASCII кода на 'A')
printf("%c\n", c + 1);       // извежда 'B' (следващата буква)
```

### 5.3. Операции с `char`

Допустими са всички целочислени операции (`+`, `-`, `*`, `/`, `%`), както и сравнение:

```c
char c = 'g';
if (c >= 'a' && c <= 'z') {
    printf("Малка буква\n");
}
```

**Преобразуване главна ↔ малка буква** (разликата между `'a'` и `'A'` в ASCII е 32):

```c
char upper = 'b' - 32;    // 'B'
char lower = 'B' + 32;    // 'b'
```

### 5.4. Въвеждане и извеждане на данни от символен тип

```c
#include <stdio.h>

int main(void) {
    char c;

    printf("Въведете символ: ");
    scanf("%c", &c);         // %c - спецификатор за char, & - адрес

    printf("Въведохте: %c (код %d)\n", c, c);
    return 0;
}
```

Функциите `getchar()` и `putchar()` също работят с по един символ:

```c
char c = getchar();   // прочита един символ от клавиатурата
putchar(c);            // извежда един символ на екрана
```

## 6. Оператор за многовариантен избор (`switch`)

> **От конспекта:** Оператор за многовариантен избор (switch) - синтаксис и начин на изпълнение.

### 6.1. Синтаксис

```c
switch (израз) {
    case константа1:
        оператори1
        break;
    case константа2:
        оператори2
        break;
    ...
    default:
        оператори_по_подразбиране
}
```

### 6.2. Начин на изпълнение

1. Изчислява се `израз` (трябва да е от целочислен тип, вкл. `char`).
2. Стойността се сравнява последователно с константите след всяко `case`.
3. При съвпадение изпълнението започва **от съответния `case`** и продължава надолу, докато не срещне `break` (или края на `switch`).
4. Ако няма съвпадение с никой `case`, изпълнява се клонът `default` (ако го има).

**Важно:** без `break` изпълнението "пропада" (*fall-through*) в следващия `case` – затова обикновено всеки клон завършва с `break`.

```c
#include <stdio.h>

int main(void) {
    int den;
    printf("Ден от седмицата (1-7): ");
    scanf("%d", &den);

    switch (den) {
        case 1: printf("Понеделник\n"); break;
        case 2: printf("Вторник\n");    break;
        case 3: printf("Сряда\n");      break;
        case 4: printf("Четвъртък\n");  break;
        case 5: printf("Петък\n");      break;
        case 6: printf("Събота\n");     break;
        case 7: printf("Неделя\n");     break;
        default: printf("Невалиден ден\n");
    }
    return 0;
}
```

**Пример за връщане на резултат от функция чрез `switch`** (от конспекта, задача с линейно уравнение `ax+b=0`):

```c
switch (uravn(a, b, &x)) {         // uravn() връща 1, 2 или 3
    case 1: printf("Безброй решения"); break;
    case 2: printf("Няма решение");    break;
    case 3: printf("x=%f", x);         break;
}
```

**`switch` срещу вложени `if`:** `switch` е по-четим, когато сравняваме **една** променлива с много конкретни стойности; `if`/`else if` е по-подходящ при сравнение на диапазони (`n > 10`) или комбинирани условия (`a > b && b > c`).

## 7. Функции

> **От конспекта:** Функции. Дефиниране и извикване на функции. Параметри на функции. Връщане на стойност от функция.

### 7.1. Определение

**Функцията** е именувана, относително самостоятелна част от програмата, която се дефинира веднъж и може да се извиква за изпълнение много пъти.

### 7.2. Дефиниране на функция

Състои се от **заглавие** и **тяло**:

- заглавие: тип на връщаната стойност, име на функцията, списък от **формални параметри** в кръгли скоби;
- тяло: `{ оператор1 оператор2 ... операторN }`

```c
float stepen(float a, int n) {     // заглавие: връща float, приема a (float) и n (int)
    float P = 1;
    int m, i;
    if (n > 0) m = n; else m = -n;
    for (i = 1; i <= m; i++)
        P = P * a;
    if (n < 0) P = 1 / P;
    return P;                       // връщане на резултата
}
```

### 7.3. Извикване на функция

Функцията се извиква с името си и списък от **фактически параметри**, чийто брой и типове трябва да съответстват на формалните параметри.

```c
int main(void) {
    float a = 2, b = 3, s;
    int n = 3, k = 2;

    s = stepen(a, n) + stepen(b, k);   // извикване на функцията два пъти
    printf("s=%f\n", s);
    return 0;
}
```

### 7.4. Параметри на функции (предаване по стойност)

В C параметрите се предават **по стойност** – функцията получава **копие** на стойността на фактическия параметър, не самата променлива.

```c
void change(int a, int b) {   // a, b са копия на x, y от main()
    int c;
    c = a; a = b; b = c;      // размяна на копията - x и y НЕ се променят
}

int main(void) {
    int x = 12, y = 10;
    change(x, y);
    printf("x=%d y=%d\n", x, y);   // извежда x=12 y=10 (без промяна!)
    return 0;
}
```

**За да разменим наистина x и y**, трябва да подадем **адресите** им с указатели:

```c
void change(int *pa, int *pb) {   // pa, pb - указатели към int
    int c;
    c = *pa;      // *pa - стойността, сочена от pa
    *pa = *pb;
    *pb = c;
}

int main(void) {
    int x = 12, y = 10;
    change(&x, &y);              // подаваме адресите на x и y
    printf("x=%d y=%d\n", x, y); // извежда x=10 y=12
    return 0;
}
```

- **Указател (pointer)** – променлива, чиято стойност е адрес от паметта.
- **`&x`** – адрес на променливата `x`.
- **`*p`** – стойността, сочена от указателя `p` (разадресиране).

### 7.5. Връщане на стойност от функция

`return израз;` прекратява изпълнението на функцията и връща стойността на `израз` на мястото на извикването. При функции от тип `void` се пише само `return;` (или изобщо не се пише).

### 7.6. Рекурсивни функции (допълнение от конспекта)

**Рекурсивна функция** – функция, която извиква себе си.

```c
int factr(int n) {
    if (n == 0) return 1;             // дъно на рекурсията (прост случай)
    else return n * factr(n - 1);     // n! = n * (n-1)!
}
```

**Бележка за производителност:** рекурсивните функции обикновено са **по-бавни** и ползват **повече памет** от итеративните (всяко извикване заделя памет за параметрите в стека), защото при всяко извикване се пази допълнителна информация в стека на извикванията, докато итеративното решение (с цикъл) използва фиксирана памет. Препоръка: за прости задачи (факториел, сума, Фибоначи) е по-ефективно да се използва цикъл; рекурсия си заслужава, когато прави решението **значително по-просто и разбираемо** (напр. работа с дървовидни структури), въпреки цената по бързина и памет.

## 8. Масиви

> **От конспекта:** Масиви - определение, размерност на масив. Дефиниране на едномерни и двумерни масиви. Достъп до елементите на едномерен и двумерен масив. Операции, въвеждане и извеждане на елементите на масив.

### 8.1. Определение

**Масивът** е структура от данни, състояща се от **фиксиран брой еднотипни елементи**.

### 8.2. Размерност на масив

Определя се от броя индекси, с които се избира елемент:

- **Едномерен масив** – всеки елемент се избира с **един** индекс.
- **Двумерен масив** – всеки елемент се избира с **два** индекса.
- **n-мерен масив** – всеки елемент се избира с `n` индекса.

### 8.3. Дефиниране и достъп – едномерен масив

```c
int A[10];   // int - тип на елементите, A - име, 10 - брой елементи (40 байта общо)
```

При дефинирането се заделя памет за всички елементи наведнъж. Индексите в C **започват от 0**: `A[0]` е първият елемент, `A[9]` е последният (10-ти) елемент.

### 8.4. Операции с елементите на масив

Операциите се извършват **поелементно**, не с масива като цяло. Всички операции, допустими за обикновена променлива от даден тип, са допустими и за елемент от масив от този тип.

### 8.5. Въвеждане и извеждане – пример (температури за n дни)

```c
#include <stdio.h>
#define max_br 100

int main() {
    float temp[max_br];
    float s = 0, avr;
    int n, i;

    do {
        printf("n=? (n<=%d): ", max_br);
        scanf("%d", &n);
    } while (n < 1 || n > max_br);

    for (i = 0; i < n; i++) {                 // въвеждане на елементите
        printf("temp за ден %d=", i + 1);
        scanf("%f", &temp[i]);
    }

    for (i = 0; i < n; i++)                   // сумиране
        s += temp[i];
    avr = s / n;
    printf("\navr=%.2f", avr);

    for (i = 0; i < n; i++)                   // отклонение от средното
        printf("\nотклонение за ден %d=%.2f", i + 1, temp[i] - avr);

    return 0;
}
```

### 8.6. Дефиниране и достъп – двумерен масив

```c
int marks[30][20];   // 30 реда (ученици) x 20 колони (предмети)
```

Всеки елемент се избира с два индекса: `marks[i][k]` – ред `i`, колона `k` (и двата започват от 0).

**Пример (съкратен, от конспекта):** оценки на ученици по предмети, среден успех на ученик, среден успех на класа по предмет, общ среден успех:

```c
#include <stdio.h>
#define max_u 30
#define max_p 20

int main() {
    int marks[max_u][max_p];
    int n, m, i, k;

    // ... въвеждане на n ученика, m предмета и оценките marks[i][k] ...

    float avr_u[max_u];                   // среден успех на всеки ученик
    for (i = 0; i < n; i++) {
        avr_u[i] = 0;
        for (k = 0; k < m; k++)
            avr_u[i] = avr_u[i] + marks[i][k];
        avr_u[i] = avr_u[i] / m;
    }

    float avr_p[max_p];                   // среден успех на класа по предмет
    for (k = 0; k < m; k++) {
        avr_p[k] = 0;
        for (i = 0; i < n; i++)
            avr_p[k] = avr_p[k] + marks[i][k];
        avr_p[k] = avr_p[k] / n;
    }

    return 0;
}
```

**Бележка за производителност:** достъп до елемент на масив (`A[i]` или `marks[i][k]`) е операция с постоянна сложност **O(1)** – независимо от размера на масива, защото адресът на елемента се изчислява директно (начален адрес + индекс × размер на елемент). Обхождане на целия едномерен масив е **O(n)**, а на двумерен `n×m` масив е **O(n·m)** – затова вложените цикли за таблицата с оценки са неизбежни, но не могат да се избегнат за задача, изискваща да се посети всеки елемент.

## 9. Символни низове

> **От конспекта:** Символни низове - определение, дефиниране. Достъп до елементите на низ. Основни функции за работа със символни низове.

### 9.1. Определение

**Символен низ (string)** – последователност от символи, завършваща с **нулев байт** `'\0'` (ASCII код 0). В C няма отделен тип `string` – низът е масив от `char`.

### 9.2. Дефиниране

```c
char s[10];    // заделят се 10 байта (вкл. 1 байт за '\0')
```

Пример: за низа `"Hello"` (5 символа) трябват минимум 6 байта (5 букви + `'\0'`).

**С начална стойност:**

```c
char s[10] = "Hello";                              // компилаторът сам добавя '\0'
char s2[10] = {'H','e','l','l','o','\0'};           // символ по символ, явно с '\0'
char s3[] = "Hello";                                // размерът (6) се подразбира от инициализацията
```

### 9.3. Достъп до елементите на низ

Достъпът е **поелементен**, с индекс, както при масив (индексите започват от 0):

```c
char s[] = "Hello";
s[0] = 'h';               // променя първата буква
printf("%s", s);          // извежда: hello
```

### 9.4. Операции с низове

**а) Въвеждане:**

```c
char s[80];
scanf("%s", s);          // въвежда до първия празен символ (интервал, табулация, нов ред)
gets(s);                  // въвежда до нов ред (остаряла, опасна функция - да се избягва)
fgets(s, 80, stdin);      // въвежда до нов ред, най-много 79 символа + '\0' - по-безопасна
```

**б) Извеждане:**

```c
printf("%s", s);
puts(s);
```

### 9.5. Основни функции за работа със символни низове (`<string.h>`)

| Функция | Действие |
|---|---|
| `strlen(s)` | връща дължината на низа `s` (без `'\0'`) |
| `strcpy(s1, s2)` | копира съдържанието на `s2` в `s1` |
| `strcat(s1, s2)` | добавя (конкатенира) `s2` в края на `s1` |

```c
#include <stdio.h>
#include <string.h>

int main() {
    char s[80] = "Hello", a[80];
    int l;

    l = strlen(s);            // l = 5
    strcpy(a, s);              // a получава копие на "Hello"
    strcat(a, " World");       // a става "Hello World"

    printf("length=%d, a=%s\n", l, a);
    return 0;
}
```

**Бележка за производителност:** `strlen`, `strcpy` и `strcat` обхождат низа символ по символ до `'\0'`, затова са със сложност **O(n)** по дължината на низа `n` (а `strcat` е O(дължина на s1 + дължина на s2), тъй като първо трябва да намери края на `s1`). При многократно извикване на `strlen` върху един и същ низ в цикъл е по-ефективно да се изчисли дължината **веднъж** и да се запази в променлива, вместо да се преизчислява на всяка итерация.

## 10. Структури

> **От конспекта:** Структури - определение, дефиниране. Достъп до елементите на структура. Масив от структури.

### 10.1. Определение

**Структурата** е съставен (потребителски) тип данни, който групира **няколко полета от различни типове** под едно име. За разлика от масива (еднотипни елементи), полетата на структурата могат да са от **различни типове**.

### 10.2. Дефиниране

```c
struct Student {          // struct - ключова дума, Student - име на структурния тип
    char name[30];         // поле от тип масив от char
    int age;               // поле от тип int
    float avgGrade;        // поле от тип float
};

struct Student s1;         // дефиниране на променлива s1 от тип struct Student
```

Може да се даде и начална стойност (поред на полетата):

```c
struct Student s1 = {"Иван", 15, 5.50};
```

### 10.3. Достъп до елементите (полетата) на структура

Достъпът е чрез **оператора точка** (`.`): `променлива.поле`

```c
#include <stdio.h>

struct Student {
    char name[30];
    int age;
    float avgGrade;
};

int main() {
    struct Student s1 = {"Иван", 15, 5.50};

    printf("Име: %s\n", s1.name);          // достъп до поле name
    printf("Възраст: %d\n", s1.age);        // достъп до поле age
    s1.avgGrade = 5.75;                     // промяна на поле avgGrade

    printf("Успех: %.2f\n", s1.avgGrade);
    return 0;
}
```

### 10.4. Масив от структури

Позволява да съхраним данни за много "обекти" от един и същ вид (напр. всички ученици в клас) в една структура от данни.

```c
#include <stdio.h>
#define MAX_STUDENTS 30

struct Student {
    char name[30];
    int age;
    float avgGrade;
};

int main() {
    struct Student klas[MAX_STUDENTS];   // масив от 30 структури
    int n, i;

    printf("Брой ученици: ");
    scanf("%d", &n);

    for (i = 0; i < n; i++) {
        printf("Име на ученик %d: ", i + 1);
        scanf("%s", klas[i].name);        // достъп до поле на елемент от масива
        printf("Успех: ");
        scanf("%f", &klas[i].avgGrade);
    }

    float sum = 0;
    for (i = 0; i < n; i++)
        sum += klas[i].avgGrade;

    printf("Среден успех на класа: %.2f\n", sum / n);
    return 0;
}
```

**Бележка за производителност:** структурата се пази в паметта като **непрекъснат блок**, съдържащ всичките полета едно след друго (понякога с малки "празнини" – *padding* – заради подравняване на данните в паметта, за по-бърз достъп). Достъп до поле на структура (`s1.age`) е операция със сложност **O(1)**, също като достъпа до елемент на масив. Обхождане на масив от `n` структури (`klas[i].avgGrade`) е **O(n)**, аналогично на обикновен масив.

## Съкращения

- **ASCII (American Standard Code for Information Interchange)** – стандартна таблица, съпоставяща на всеки символ (буква, цифра, знак) числов код от 0 до 127.
- **ОП** – оперативна памет; частта от паметта на компютъра, в която се съхраняват данните и програмите по време на изпълнение.
- **C** – език за програмиране (не е абревиатура, а буквено име на езика), използван в този курс.
- **O(n), O(1), O(√n)** – нотация за сложност на алгоритъм ("Big O"); показва как расте времето/паметта на изпълнение при увеличаване на размера на входа `n`. `O(1)` означава постоянно време, `O(n)` – право пропорционално на `n`, `O(√n)` – пропорционално на квадратния корен от `n`.
- **IDE (Integrated Development Environment)** – интегрирана среда за разработка (тук не се използва пряко, но е споменато за пълнота при работа с код).
