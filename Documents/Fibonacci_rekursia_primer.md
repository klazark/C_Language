# Елементарен пример: n-тото число на Фибоначи с рекурсия (език C)

Този текст е за начинаещ: обяснява **какво** правим, **защо** работи рекурсията и **как** да го напишем в C стъпка по стъпка.

---

## 1. Какво е редицата на Фибоначи (Fibonacci sequence)?

Започваме с две „начални“ числа, обикновено **0** и **1**. Всяко следващо число е **сумата** от предходните две:

- F(0) = 0  
- F(1) = 1  
- F(2) = F(1) + F(0) = 1 + 0 = 1  
- F(3) = F(2) + F(1) = 1 + 1 = 2  
- F(4) = 2 + 1 = 3  
- F(5) = 3 + 2 = 5  
- …

Така получаваме: **0, 1, 1, 2, 3, 5, 8, 13, …**

**Важно за курса:** понякога в задачи се ползва вариант с F(1)=1, F(2)=1 без нулата. Винаги проверявайте **условието** кое начало изисква. Тук ползваме стандартното **F(0)=0, F(1)=1**.

---

## 2. Какво е рекурсия (recursion)?

**Рекурсия** означава: **функцията извиква сама себе си**, но при правилно написан код всяко извикване се „приближава“ към **краен случай**, при който **спираме** да се викаме отново.

За да е коректна една рекурсивна функция, обикновено са нужни две неща:

1. **Базов случай (base case)** – при малко n връщаме **готова стойност** без ново извикване (напр. F(0) и F(1)).  
2. **Рекурсивна стъпка (recursive step)** – за по-голямо n изразяваме резултата чрез **същата функция** с **по-малки** аргументи (тук: n−1 и n−2).

Ако няма базов случай (или той никога не се достига), програмата може да „върти“ безкрайно и да свърши с **stack overflow** (препълване на стека за извиквания) – това е грешка по време на изпълнение.

---

## 3. Математическата идея (рекурсивна дефиниция)

За n ≥ 2:

**F(n) = F(n−1) + F(n−2)**

А за малките n:

- **F(0) = 0**  
- **F(1) = 1**

Това е точно „шаблонът“, който ще прехвърлим в C.

---

## 4. Прост план (стъпки преди кода)

| Стъпка | Какво правим |
|--------|----------------|
| **1** | Решаваме какво връща функцията: тук **`int`** (за малки n е достатъчно; за големи n целите числа растат бързо – виж раздел 8). |
| **2** | Избираме име: напр. `fibonacci`. |
| **3** | Параметър: **`int n`** – кое число от редицата искаме. |
| **4** | Пишем **базовите случаи**: ако `n == 0` връщаме 0; ако `n == 1` връщаме 1. |
| **5** | Пишем **рекурсивната част**: `return fibonacci(n - 1) + fibonacci(n - 2);` |
| **6** | В `main` четем `n`, викаме функцията, печатаме резултата. |
| **7** | Тестваме с малки n (0, 1, 2, 3, 4, 5) и сравняваме с ръчно пресметнати стойности. |

---

## 5. Стъпка по стъпка: как се чете изпълнението (пример n = 4)

Когато извикаме `fibonacci(4)`, езикът прави нещо като „дърво“ от извиквания:

- `fibonacci(4)` се нуждае от `fibonacci(3)` + `fibonacci(2)`  
- `fibonacci(3)` се нуждае от `fibonacci(2)` + `fibonacci(1)`  
- … докато не стигнем до `fibonacci(1)` → **1** и `fibonacci(0)` → **0**

Накрая всички „къси“ резултати се събират нагоре по дървото и получаваме **F(4) = 3**.

Това **не** е магия: компилаторът запазва **къде** сме спрели всяко извикване и след като подизвикването върне стойност, продължава от там. Това се държи в **стека на извикванията (call stack)**.

---

## 6. Пълен пример на програма на C

```c
#include <stdio.h>

/* Декларация (прототип): казваме на компилатора, че такава функция съществува. */
int fibonacci(int n);

int main(void)
{
    int n;

    printf("n = ");
    if (scanf("%d", &n) != 1) {
        printf("Невалиден вход.\n");
        return 1;
    }

    if (n < 0) {
        printf("За този пример n трябва да е >= 0.\n");
        return 1;
    }

    printf("F(%d) = %d\n", n, fibonacci(n));
    return 0;
}

/*
 * Връща n-тото число на Фибоначи при F(0)=0, F(1)=1.
 */
int fibonacci(int n)
{
    /* Стъпка A: базови случаи – спираме рекурсията */
    if (n == 0) {
        return 0;
    }
    if (n == 1) {
        return 1;
    }

    /* Стъпка B: рекурсивна стъпка – задачата става по-малка */
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

**Какво да запомни ученикът:**

- `return` **прекратява** текущото извикване и **връща** стойност нагоре.  
- При `n >= 2` всяко извикване прави **две** нови извиквания – затова броят работа расте много бързо (виж по-долу).

---

## 7. Чести грешки

1. **Липса на базов случай** – безкрайна рекурсия.  
2. **Грешен базов случай** – напр. само `n == 1`, а нулата не е обработена.  
3. **Грешно условие** – объркани F(0)/F(1) според условието на задачата.  
4. **Много голямо n** – програмата става бавна или числото не се събира в `int`.

---

## 8. Производителност: защо „наивната“ рекурсия е бавна и какви са вариантите

Наивната форма `fibonacci(n-1) + fibonacci(n-2)` изчислява **много пъти едни и същи** подпроблеми (напр. `fibonacci(2)` се появява отнова и отново в дървото).

### Варианти (от бавен към по-бърз за големи n)

| Подход | Идея | Плюсове | Минуси |
|--------|------|---------|--------|
| **Наивна рекурсия** | Директно от формулата | Много кратък код, добър за обучение | Време расте експоненциално – за големи n е непрактична |
| **Итерация (цикъл)** | Запазваме последните две стойности в променливи | Бързо, O(n) на време, малко памет | По-дълъг код от рекурсията |
| **Рекурсия + запомняне (мемоизация, memoization)** | Кешираме вече пресметнати F(k) | Бързо O(n), запазва духа на рекурсията | Нужен е масив или речник (map); повече памет |
| **Матрично повдигане на степен** | Математически трик за F(n) | Много бързо за огромни n | Сложно за начинаещ курс |

**Препоръка за курс „Увод в програмиране“:**  
- За **разбиране на рекурсия** – наивната версия е идеална за малки n (напр. n ≤ 30–35, зависи от машината).  
- За **реална употреба** – след като разберете рекурсията, научете **итеративния** вариант; той е стандартният „бърз и прост“ начин.

**Забележка за типове:** `int` обикновено е32 бита и преливането (overflow) настъпва още при умерено големи n. За по-големи числа се ползват `long long` или библиотеки за големи цели числа – извън този елементарен пример.

---

## 9. Кратък речник на съкращения и термини

- **F(n)** – n-тото число на Фибоначи в избраното начало на редицата (тук F(0)=0, F(1)=1).  
- **Рекурсия (recursion)** – техника, при която функцията извиква сама себе си с по-малък/опростен аргумент до достигане на базов случай.  
- **Базов случай (base case)** – условие, при което функцията връща директен отговор без ново самоизвикване.  
- **Стек на извикванията (call stack)** – структура, в която програмата помни „къде“ да продължи след завръщане от функция; при много нива рекурсия стекът може да се препълни (**stack overflow**).  
- **O(n) (Big O)** – нотация за оценка как расте времето (или паметта) при увеличаване на n; O(n) означава „приблизително пропорционално на n“.  
- **O(2^n) (експоненциална сложност)** – наивната рекурсия за Фибоначи е от този тип: удвояване на работата при увеличаване на n с малко.  
- **Мемоизация (memoization)** – запазване на вече изчислени резултати, за да не ги смятаме повторно.  
- **CPU (Central Processing Unit)** – централен процесор; изпълнява инструкциите на програмата; при бавен алгоритъм CPU работи дълго върху същата задача.  
- **PCI DSS (Payment Card Industry Data Security Standard)** – стандарт за защита на данни от плащания с карти; **не се отнася** до учебната програма за Фибоначи; споменава се само ако същият код по някаква причина обработва чувствителни платежни данни (което тук не е случаят).
