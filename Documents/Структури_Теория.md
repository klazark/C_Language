Структури (struct) в C — теория и примери

1. Какво е структура?

Структурата е съставен (структуриран) тип данни, който групира под едно име няколко променливи (наречени полета или членове), които могат да бъдат от различни типове. За разлика от масива (който съдържа елементи само от един тип), структурата позволява да опишем обект с разнородни характеристики — например ученик с номер (цяло число), име (низ) и успех (реално число).

2. Дефиниране на структурен тип

struct Student {
    int number;
    char name[30];
    float grades[20];
    float average;
};

Тук struct Student е новият тип. Самото дефиниране не заделя памет — то само описва "шаблона".

3. Деклариране на променливи от структурен тип

struct Student s1;          // една променлива
struct Student class[30];   // масив от структури (30 ученика)

4. Достъп до полетата — операторът .

s1.number = 1;
strcpy(s1.name, "Ivan");
s1.grades[0] = 5.5;
s1.average = 5.5;

printf("%d %s %.2f", s1.number, s1.name, s1.average);

5. Инициализация при дефиниране

struct Student s2 = {2, "Maria", {6, 5, 6}, 5.67};

6. typedef — по-удобно име на типа

Без typedef винаги трябва да пишем struct Student. С typedef можем да съкратим до Student:

typedef struct {
    int number;
    char name[30];
    float average;
} Student;

Student s1;   // вместо struct Student s1;

7. Масив от структури

#define MAX_STUDENTS 30

typedef struct {
    int number;
    char name[30];
    float grades[20];
    float average;
} Student;

int main() {
    Student class_[MAX_STUDENTS];
    int n, m, i, k;

    printf("n=? "); scanf("%d", &n);   // брой ученици
    printf("m=? "); scanf("%d", &m);   // брой предмети

    for (i = 0; i < n; i++) {
        class_[i].number = i + 1;
        printf("name %d: ", i + 1);
        scanf("%s", class_[i].name);

        float sum = 0;
        for (k = 0; k < m; k++) {
            printf("grade %d: ", k + 1);
            scanf("%f", &class_[i].grades[k]);
            sum += class_[i].grades[k];
        }
        class_[i].average = sum / m;
    }

    for (i = 0; i < n; i++)
        printf("%-3d %-10s %.2f\n", class_[i].number, class_[i].name, class_[i].average);

    return 0;
}

Забележете разликата с масив от int: class_[i] тук е цяла структура, а class_[i].average е нейно поле.

8. Влагане на структури (nested structures)

Една структура може да съдържа друга структура като поле:

typedef struct {
    int day, month, year;
} Date;

typedef struct {
    int number;
    char name[30];
    Date birth_date;
} Student;

Student s;
s.birth_date.year = 2005;   // достъп през две точки

9. Указатели към структури — операторът ->

Student s1 = {1, "Ivan", {2005, 5, 10}};
Student *p = &s1;

printf("%d", (*p).number);   // еквивалентно на p->number
printf("%d", p->number);     // по-удобен запис

Това е важно, защото предаването на структури на функции по стойност копира цялата структура (бавно за големи структури). Затова често се предават указатели:

void print_student(Student *p) {
    printf("%d %s %.2f\n", p->number, p->name, p->average);
}

void set_average(Student *p, int m) {
    float sum = 0;
    for (int k = 0; k < m; k++)
        sum += p->grades[k];
    p->average = sum / m;   // променя оригинала, не копие
}

10. Сравнение: масив срещу структура срещу масив от структури

Характеристика: Тип на елементите
- Масив: Един и същ тип
- Структура: Различни типове полета
- Масив от структури: Всеки елемент е една структура

Характеристика: Достъп
- Масив: a[i]
- Структура: s.field
- Масив от структури: arr[i].field

Характеристика: Типична употреба
- Масив: Списък от еднородни стойности (оценки, температури)
- Структура: Един обект с няколко характеристики (ученик, дата)
- Масив от структури: Списък от обекти (клас ученици)

11. Анализ на производителността

- Предаване по стойност срещу указател: предаването на голяма структура (напр. с масив от 20 оценки) по стойност копира цялата памет при всяко извикване на функция — това е разход по време и памет, пропорционален на размера на структурата. Предаването на указател (Student *) копира само адреса (обикновено 8 байта на 64-битова система) и затова е предпочитаният подход за големи структури или когато функцията трябва да променя оригинала.
- Подравняване на паметта (padding): компилаторът може да добавя "празни" байтове между полетата на структурата, за да ги подравни в паметта (напр. int след char често се подрежда на граница, кратна на 4). Това увеличава размера на sizeof(struct) спрямо простата сума на полетата. За учебни задачи (масив до 30 ученика) това е незначително, но при масиви от милиони структури може да има осезаем ефект върху кеш паметта и разхода на RAM (оперативна памет с произволен достъп).
- Масив от структури срещу структура от масиви: за задачата с 30 ученика x 20 оценки подходът "масив от структури" (Array of Structs) е по-четим и достатъчно бърз. При много по-големи обеми данни и чести операции само върху едно поле (например само оценките), "структура от масиви" (Struct of Arrays) би била по-кеш-приятелска, защото данните за едно поле лежат последователно в паметта. За учебна задача (n<=30, m<=20) разликата е практически нулева — препоръката е да се остане с по-простия и четим вариант "масив от структури".
