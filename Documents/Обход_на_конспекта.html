§Основни понятия и синтаксис

Тук са събрани универсалните определения на думите, които се използват навсякъде по-долу. Ако разбереш тези понятия, всеки следващ пример се чете лесно.

Израз, оператор, блок

Израз (expression)
Комбинация от стойности, променливи, операции и извиквания на функции, която се пресмята до една стойност. Примери: 3 + 4, a * b, grade >= 60, n % 10, stepen(a, n). В условие изразът се тълкува като истина (различно от 0) или лъжа (0).
Оператор (statement)
Една завършена команда (стъпка), която компютърът изпълнява. Обикновено завършва с точка и запетая ;. Примери: a = 5; (оператор за присвояване), printf("Hi"); (извикване), if (...) ... (условен оператор), while (...) ... (оператор за цикъл). Там, където схемата пише оператор1/оператор2, се има предвид една такава команда — или единичен оператор, или цял блок { }.
Съставен оператор / блок
Няколко оператора, заградени във фигурни скоби { }, които се третират като един оператор. Използва се в if, else, while, do-while, for, тялото на функция и т.н.
Инициализация
Задаване на начална стойност на променлива. Напр. int i = 1; или частта i = 1 в заглавието на for.
Инкрементация / стъпка
Инкрементация = увеличаване с 1 (i++ е същото като i = i + 1); декрементация = намаляване с 1 (i--). В заглавието на for това е стъпката — как се променя броячът на всяко повторение.
Итерация
Едно завъртане (едно изпълнение на тялото) на цикъл.

Директиви на предпроцесора (редове с #)

Директива е ред, който започва с # и се обработва от предпроцесора — стъпка, която се изпълнява преди същинската компилация. Директивата не е обикновен оператор: не завършва с точка и запетая ; и обикновено стои в началото на файла. Двете най-важни са:

ДирективаКакво прави
#include„Вмъква" съдържанието на заглавен файл (библиотека) в програмата — така получаваш достъп до готови функции. <stdio.h> дава printf/scanf; <string.h> — strlen/strcpy; <math.h> — математически функции. Ъгловите скоби < > означават стандартна (системна) библиотека.
#defineЗадава символна константа (макрос): преди компилацията предпроцесорът заменя всяко срещане на името със стойността. #define N 30 прави така, че всяко N в кода става 30. По конвенция имената се пишат с главни букви.
#include <stdio.h>    // за printf и scanf
#define N 30          // символна константа (без = и без ;)

int main() {
    int a[N];             // след замяната -> int a[30];
    printf("Готово\n");
    return 0;
}
Защо #define? Ако размерът е записан на едно място като N, промяната му (напр. от 30 на 40) става с една редакция, вместо да търсиш всяко 30 из целия код. Внимание: това е чиста текстова замяна — след реда с #define няма ; (иначе и точката и запетаята ще влезе в заместването).

Кога се слагат фигурни скоби и кога може да се пропуснат

След if, else, while, for „принадлежи“ само един оператор. Ако тялото е един-единствен оператор, фигурните скоби може да се пропуснат. Ако операторите са два или повече — скобите са задължителни, за да ги обединят в блок.

if (x > 0)
    printf("положително");   // 1 оператор -> скоби не са нужни

if (x > 0) {
    y = x * 2;                // 2 оператора -> скобите
    printf("%d", y);          // СА задължителни
}
Съвет: дори при един оператор мнозина винаги слагат { } — така по-късно е лесно да добавиш втори ред, без да сгрешиш. Класически капан е следният код: 
if (x > 0)
    printf("A");
    printf("B");   // НЕ е в if-а! Изпълнява се ВИНАГИ (отстъпът лъже)
Отстъпът (indentation) е само за човека — компилаторът гледа скобите, не празните места.

Референция и достъп до елементи / полета

Референция (обръщение) означава „да посочиш“ конкретна стойност или част от съставен обект, за да я прочетеш или промениш. В зависимост от типа данни се използват различни оператори:

ЗаписИме на оператораКакво прави
A[i]индексиране (квадратни скоби)достъп до елемент на масив по индекс (брои се от 0).
s.nameточка .достъп до поле на структура през самата променлива.
p->nameстрелка ->достъп до поле на структура през указател към нея; p->name е кратък запис на (*p).name.
&aамперсанд & (адрес-на)взима адреса на променливата a (нужен на scanf и на указатели).
*pзвездичка * (стойност-по-адрес)стойността, която се намира на адреса в указателя p.

Тези три реда достъпват едно и също поле по три начина:

s.average = 5.5;        // през променливата (точка)
class_[i].average = 5.5;   // елемент i от масив, после негово поле
p->average = 5.5;         // през указател (стрелка)

Прост пример с & (адрес-на) и * (стойност-по-адрес)

Аналогия: представи си, че всяка променлива е апартамент. Адресът е номерът на апартамента (къде се намира), а стойността е кой живее вътре. Тогава:

  • &a пита: „Кой е номерът на апартамента a?" → връща адрес.
  • *p пита: „Кой живее на адреса, записан в p?" → връща стойността там.
int a = 5;
int *p = &a;          // p пази АДРЕСА на a (p "сочи" към a)

printf("%d\n", a);     // 5              - стойността на a
printf("%p\n", &a);    // 0x7ffee3b0c4ac - АДРЕСЪТ на a (къде е в паметта)
printf("%p\n", p);     // 0x7ffee3b0c4ac - същият адрес (p сочи натам)
printf("%d\n", *p);    // 5              - стойността НА адреса в p (обратно към a)

*p = 9;               // променяме стойността ПРЕЗ указателя
printf("%d\n", a);     // 9              - a СЕ ПРОМЕНИ, защото p сочи към a!

Какво реално виждаш като стойност:

ИзразКакво еПримерна стойност на екрана
aстойността на променливата5
&aадресът на a (шестнайсетично число)0x7ffee3b0c4ac
pадресът, който указателят пази (същият)0x7ffee3b0c4ac
*pстойността, намираща се на този адрес5
Двата оператора са обратни: & тръгва от променлива и дава адрес; * тръгва от адрес и дава стойност. Затова *(&a) е просто a. Ефектът, който виждаш: адресите (&a, p) са дълги hex числа, които се сменят при всяко пускане на програмата; стойностите (a, *p) са нормалните числа. Промяна през *p се отразява върху a, защото двете сочат едно и също място в паметта. Точно затова scanf("%d", &a) ползва & — дава на scanf адреса, за да може да запише в a.

Колко параметъра приемат scanf и printf?

printf и scanf са функции с променлив брой параметри (variadic). Правилото е просто:

Първи параметър — форматен низ
Винаги точно един низ в кавички: за printf това е текстът за извеждане, за scanf — какви данни да прочете. В него стоят форматни спецификатори: %d (int), %f (float), %lf (double), %c (char), %s (низ).
Следващи параметри — по един за всеки спецификатор
Броят им не е фиксиран — колкото %-спецификатора има във форматния низ, толкова аргумента трябва да подадеш след него (в същия ред). Може да са 0, 1, 2, 3 или повече.
printf("Hello\n");                 // само форматен низ (0 допълнителни)
printf("%d", a);                     // 1 спецификатор -> 1 аргумент
printf("%s е на %d г.\n", name, age); // 2 спецификатора -> 2 аргумента

scanf("%d %d", &a, &b);            // 2 стойности -> 2 адреса (&)
Важно за scanf: след форматния низ се подават адреси (&a), защото функцията трябва да запише в променливите. Изключение са низовете (char масив), при които името вече е адрес: scanf("%s", s) — без &. Ако броят на спецификаторите и на аргументите не съвпада, програмата се държи непредсказуемо.

2Реални типове данни

Константи: с десетична точка (2.33, -12.4) или с експонента (2.5E-1 = 2.5·10⁻¹).

Променливи: float a; (4 байта), double b; (8 байта) — разлика в точността и диапазона.

Типът double — подробно

double („двойна точност", от double precision) е реален тип, който заема 8 байта (64 бита). Съхранява числа с дробна част в „плаваща запетая" — знак, мантиса (значещите цифри) и експонента (порядъка). В сравнение с float той пази повече значещи цифри и покрива по-голям диапазон.

ТипРазмерТочност (значещи цифри)Приблизителен диапазон
float4 байта≈ 7 цифри≈ ±3.4·10³⁸
double8 байта≈ 15–16 цифри≈ ±1.7·10³⁰⁸

По подразбиране реалните константи са double: литерал като 3.14 или 2.5E-1 е от тип double. За да е float, се пише наставка f: 3.14f. Функциите от <math.h> (напр. sqrt, sin) също работят и връщат double. Затова double е обичайният избор за реални пресмятания, а float се ползва, когато паметта е критична (напр. големи масиви).

Вход/изход при double (важно!)

Тук има несиметрия, която често обърква:

Типscanf (четене)printf (печат)
float%f%f
double%lf%f (или %lf)
double b;
scanf("%lf", &b);       // за double ЗАДЪЛЖИТЕЛНО %lf ('l' = long)
printf("%f\n", b);       // при печат %f работи и за double
printf("%.2f\n", b);     // %.2f -> 2 цифри след запетаята
Внимание — реалните числа не са точни: някои стойности (напр. 0.1) не могат да се представят точно в двоичен вид, затова 0.1 + 0.2 може да даде 0.30000000000000004. Затова не сравнявай реални числа с ==; вместо това проверявай дали разликата е достатъчно малка: if (fabs(x - y) < 1e-9). double греши по-рядко от float, защото пази повече цифри, но проблемът остава по принцип.

Внимание при деление

int s = 7, n = 2;
float avr;
avr = s / n;          // ГРЕШНО - цяло/цяло = 3 (0.5 се губи)
avr = (float)s / n;   // ВЯРНО - явно преобразуване -> 3.5

Вход/изход

3Условен оператор (if/else)

Синтактична схема (шаблон)

if (израз)
    оператор1
else
    оператор2
израз
условие в кръгли скоби, което се пресмята до истина (≠ 0) или лъжа (0), напр. grade >= 60.
оператор1
изпълнява се, ако изразът е истина. Може да е един оператор или блок { }.
оператор2
изпълнява се, ако изразът е лъжа. Частта else е незадължителна — без нея имаме непълна форма.
if (grade >= 60)          // пълна форма
    printf("Взето\n");
else
    printf("Скъсано\n");

if (temp > 30)             // непълна форма (без else)
    printf("Горещо\n");

Вложени условни оператори (nested if/else)

if вътре в if (или в else) — полезно за многостепенни проверки, при които второто условие има смисъл само ако първото вече е изпълнено. Пример — намиране на най-голямото от 3 числа:

if (a > b) {
    if (a > c)                  // влизаме тук само ако a > b
        printf("Най-голямо: a = %d\n", a);
    else
        printf("Най-голямо: c = %d\n", c);
} else {
    if (b > c)                  // влизаме тук само ако b >= a
        printf("Най-голямо: b = %d\n", b);
    else
        printf("Най-голямо: c = %d\n", c);
}
Обяснение: първо сравняваме a и b, за да стесним избора до двойка кандидати; вътрешният if/else сравнява победителя с c. Всяко ниво на влагане се изпълнява само ако условието на нивото над него е било истина (или лъжа, при else клона) — по този начин с 2 нива проверки покриваме 3 числа, вместо да пишем сложен единичен израз с &&/||.

Приоритет (старшинство) на операторите

Когато в един израз има няколко оператора, те се изпълняват по приоритет (старшинство) — по-старшите първи. Ако приоритетът е еднакъв, редът обикновено е отляво надясно. Ето най-важните за условия, подредени от най-висок към най-нисък:

ПриоритетОператориОписание
1 (най-висок)( )скоби — винаги първи, „заобикалят" всички правила
2!логическо НЕ (унарен — действа на едно нещо)
3* / %умножение, деление, остатък
4+ -събиране, изваждане
5< <= > >=сравнение за наредба
6== !=сравнение за равенство/различие
7&&логическо И
8||логическо ИЛИ
9 (най-нисък)=присвояване

Прости примери (в скоби е показано как компилаторът „вижда" израза):

a + 1 > b        // (a + 1) > b     - аритметика преди сравнение
x > 0 && x < 10  // (x > 0) && (x < 10)  - сравнение преди &&
!found && ok    // (!found) && ok  - ! преди &&
a || b && c     // a || (b && c)   - && преди ||
2 + 3 == 5     // (2 + 3) == 5    - + преди ==  -> истина
Правило за оцеляване: && е по-старши от ||, а сравненията (>, ==…) са по-старши от &&/||. Затова x > 0 && x < 10 работи без скоби. Когато обаче смесваш && и ||, слагай скоби за яснота: (a || b) && c е нещо различно от a || (b && c).
↑ горе

4Оператори за цикъл

С предусловие (while)

Условието се проверява преди тялото; може да не се изпълни нито веднъж:

while (n > 0) {
    printf("%d ", n % 10);
    n = n / 10;
}

С постусловие (do-while)

Тялото се изпълнява поне веднъж, след което се проверява условието:

do {
    printf("%d\n", x);
    x++;
} while (x < 3);

Кога логично да използваш do-while

Използвай do-while, когато първо трябва да направиш нещо, а едва после да решиш дали да го повториш. Най-типичният случай е валидация на вход от потребителя — питаш поне веднъж, проверяваш, и ако е грешно, питаш отново.

// Потребителят ТРЯБВА да въведе число между 1 и 100
do {
    printf("Въведи число от 1 до 100: ");
    scanf("%d", &n);
} while (n < 1 || n > 100);   // повтаря само ако е извън диапазона

printf("Прието: %d\n", n);
Защо не while? С while трябва или да инициализираш n с невалидна стойност (напр. n = 0), или да дублираш scanf преди цикъла. С do-while логиката е естествена: направи → провери → повтори при нужда.

Друг типичен случай — меню, което се показва поне веднъж:

do {
    printf("1. Нова игра\n2. Резултат\n0. Изход\nИзбор: ");
    scanf("%d", &choice);
    // обработка на избора...
} while (choice != 0);

Цикъл for

Компактен цикъл с брояч. В заглавието му, в кръгли скоби и разделени с ;, стоят три части:

for (инициализация; условие; стъпка)
    оператор
инициализация
изпълнява се веднъж в началото — задава начална стойност на брояча, напр. i = 1.
условие
проверява се преди всяко повторение; докато е истина, тялото се изпълнява, напр. i <= 5.
стъпка (инкрементация)
изпълнява се след всяко тяло — променя брояча, напр. i = i + 1 (или i++).
оператор
тялото — единичен оператор или блок { }.

Ред на изпълнение: инициализация → (условие → тяло → стъпка) → (условие → тяло → стъпка) → … докато условието стане лъжа.

for (i = 1; i <= 5; i = i + 1)
    printf("%d ", i);   // 1 2 3 4 5

Как да избера вида цикъл?

И трите цикъла могат да свършат една и съща работа — изборът е въпрос на четимост и на това коя логика е най-естествена. Водещият въпрос е: знам ли предварително колко пъти ще се повтори?

ЦикълКога го предпочитамеТипични примери
for Знаем точно броя повторения; има брояч с фиксирана стъпка; искаме компактност (инициализация, условие и стъпка на едно място). числа от 1 до n; въвеждане на n стойности; обхождане на масив по индекс.
while Не знаем броя предварително; условието зависи от данни, които се променят в тялото; трябва да проверим преди първото изпълнение (може и 0 пъти). обработка на цифрите на число (докато стане 0); четене до специална стойност (напр. 0 или −1).
do-while Тялото трябва да се изпълни поне веднъж, а после да решим дали да повторим (проверка след тялото). валидация на въведени данни; меню, което се показва поне веднъж.
Практично правило: започни с въпроса „брояч с известен край?" → for. „Повтарям, докато нещо се случи, без да знам колко пъти?" → while. „Задължително поне веднъж?" → do-while. Технически всеки for може да се пренапише като while и обратно — избираме този, който прави кода по-ясен.

Пример за същия резултат, изразен и с двата цикъла:

// for - броячът, условието и стъпката са събрани в заглавието
for (i = 1; i <= 5; i++)
    printf("%d ", i);

// while - същото, но частите са разпръснати
i = 1;                 // инициализация - преди цикъла
while (i <= 5) {         // условие
    printf("%d ", i);
    i++;                // стъпка - в тялото
}

Вложени цикли (nested loops)

Вложен цикъл означава цикъл вътре в тялото на друг цикъл. Външният наричаме външен цикъл, вътрешния — вътрешен цикъл. Ключовото правило: за всяко едно завъртане на външния цикъл вътрешният се завърта изцяло (от начало до край).

for (i = 1; i <= 3; i++) {      // външен цикъл - 3 реда
    for (k = 1; k <= 4; k++)   // вътрешен цикъл - 4 звезди
        printf("* ");
    printf("\n");              // нов ред след всеки ред
}
* * * *
* * * *
* * * *

Тялото printf("* ") се изпълнява 3 × 4 = 12 пъти. Външният брояч i отчита редовете, вътрешният k — колоните. Затова вложените цикли са естественият избор за двумерни масиви (матрици) и таблици.

Оператор за прекратяване (break)

break прекратява цикъла веднага, без да чака условието да стане лъжа — изпълнението продължава след цикъла. Полезен е, когато отговорът е намерен по-рано, напр. при проверка дали число е просто: щом намерим един делител, няма смисъл да търсим повече.

for (i = 2; i < n; i++) {
    if (n % i == 0) {       // намерихме делител ->  n не е просто
        isPrime = 0;
        break;              // излизаме веднага от цикъла
    }
}
break прекъсва само един (най-вътрешния) цикъл, в който се намира.

5Символен тип данни. ASCII таблица

char съхранява един символ в 1 байт — реално е малко цяло число, кодирано по ASCII (American Standard Code for Information Interchange — таблица, свързваща числа 0–127 със символи, напр. 'A' = 65, 'a' = 97, '0' = 48).

char c = 'A';
printf("%c има код %d\n", c, c);   // A има код 65

Понеже char е числов тип, върху него важат аритметичните операции:

char digit = '7';
int value = digit - '0';   // '7'-'0' = 7 (символ -> число)

Вход/изход: %c за единичен символ; getchar()/putchar() четат/пишат по един символ.

6Оператор switch

switch (day) {
    case 1: printf("Понеделник"); break;
    case 2: printf("Вторник"); break;
    default: printf("Друг ден");
}

Изпълнение: сравнява израза с всяко case; при съвпадение изпълнява от там нататък. break е задължителен, за да не „пропадне“ (fall-through) в следващия case. default е незадължителен и се изпълнява при липса на съвпадение. Полезен като по-четима алтернатива на верига от if-else if.

Какво стои в скобите на switch(...)?

В скобите стои целочислен израз — променлива или израз, който се пресмята до цяло число (напр. day, choice, n % 3). Всеки етикет case е константа от същия тип и стойностите на case трябва да са различни.

Тип на изразаРазрешен в switch?
int, short, long✅ да (целочислени)
char✅ да (символът е малко цяло число по ASCII)
enum (изброен тип)✅ да (стойностите му са цели числа)
float, double❌ не (реални числа)
низ (char[] / "текст")❌ не — не може да се превключва по низ

Понеже char е целочислен тип, switch върху символ е напълно валиден и много удобен за менюта:

char op;
scanf(" %c", &op);
switch (op) {
    case '+': printf("%d\n", a + b); break;
    case '-': printf("%d\n", a - b); break;
    default: printf("Непозната операция\n");
}
Какво НЕ може в case: етикетът трябва да е константа, известна при компилация — не може да е променлива (case x:), диапазон (case 1..5:) или условие (case n > 0:). За такива случаи (реални числа, низове, диапазони) се използва верига от if / else if.

7Функции

Дефиниция

заглавие (тип + име + параметри) + тяло:

float stepen(float a, int n) {   // формални параметри
    float p = 1;
    for (int i = 1; i <= n; i++) p = p * a;
    return p;
}

Извикване: s = stepen(a, n); (фактически параметри).

Параметри по стойност

Функцията получава копие, не може да променя оригинала:

void change(int a, int b) { int c = a; a = b; b = c; }
// НЕ разменя x и y в main()

Указатели (pointers)

Променлива, съдържаща адрес — решава проблема по-горе:

void change(int *pa, int *pb) {
    int c = *pa; *pa = *pb; *pb = c;
}
change(&x, &y);   // сега x и y СЕ разменят

Рекурсия (рекурсивно извикване)

Рекурсия е когато функция извиква сама себе си, за да реши по-малка версия на същата задача. Всяка коректна рекурсия има две задължителни части:

Базов случай („дъно")
условие, при което функцията връща стойност без да се извиква повече — така рекурсията спира. При факториела това е n == 0.
Рекурсивен случай
функцията се извиква със стъпка към базовия случай (напр. n - 1) и построява отговора от резултата на по-малкото извикване.
int fact(int n) {
    if (n == 0) return 1;        // базов случай - спира рекурсията
    return n * fact(n - 1);      // рекурсивен случай - извиква себе си
}

Как се разгъва fact(3): първо извикванията се „спускат" до дъното, после стойностите се връщат нагоре:

fact(3) = 3 * fact(2)
        = 3 * (2 * fact(1))
        = 3 * (2 * (1 * fact(0)))
        = 3 * (2 * (1 * 1))          // достигнато дъно: fact(0) = 1
        = 3 * (2 * 1)
        = 3 * 2
        = 6
Внимание — безкрайна рекурсия: ако липсва базов случай или стъпката не се приближава към него (напр. fact(n) вместо fact(n - 1)), функцията се вика безспир, докато не се препълни стекът (stack overflow) и програмата „забие". Затова базовият случай е задължителен.
Рекурсия срещу цикъл: всяка рекурсия може да се напише и с цикъл (итеративно). Рекурсията е по-бавна и ползва повече памет (за всяко извикване се заделя памет в стека), но за някои задачи (факториел, Фибоначи, обхождане на дървовидни структури) кодът е по-къс и по-близък до математическата дефиниция.

float temp[100];
for (i = 0; i < n; i++) scanf("%f", &temp[i]);
for (i = 0; i < n; i++) s += temp[i];
avr = s / n;

Двумерен масив (матрица)

int marks[30][20]; — избира се с два индекса marks[i][k] (ред, колона). Използва се за таблици (напр. оценки на ученици по предмети), с вложени for цикли за обхождане.

Всички операции се извършват поелементно — не съществува „цял масив + цял масив“.

Пример: оценки на клас — средна оценка и брой отличници

Обхождаме матрицата grades[n][m] (n ученици × m предмета) с вложени цикли. Едновременно (1) сумираме всички оценки за средната оценка на класа и (2) с флаг проверяваме за всеки ученик дали има само 6-ци.

#define N 30            // максимум ученици
#define M 5             // максимум предмети

float grades[N][M];
int n, m, i, k;
float classSum = 0;      // сума на ВСИЧКИ оценки в класа
int allSixes = 0;         // брояч: ученици само с 6-ци

scanf("%d %d", &n, &m);   // брой ученици и предмети

for (i = 0; i < n; i++) {        // външен цикъл - по ученици
    int onlySix = 1;             // флаг: приемаме "да", докато не се докаже обратното
    for (k = 0; k < m; k++) {    // вътрешен цикъл - по предмети
        scanf("%f", &grades[i][k]);
        classSum += grades[i][k];  // трупаме към общата сума
        if (grades[i][k] != 6)
            onlySix = 0;         // намерена оценка != 6 -> вече не е "само 6-ци"
    }
    if (onlySix)
        allSixes++;                // този ученик има само 6-ци
}

float classAvg = classSum / (n * m);   // средна оценка за класа

printf("Средна оценка за класа: %.2f\n", classAvg);
printf("Ученици само с 6-ци: %d\n", allSixes);
Средна оценка за класа: 5.24
Ученици само с 6-ци: 3
Флаг (булев признак): onlySix започва с 1 („да") и става 0 при първата оценка, различна от 6. Този шаблон „приеми истина → опровергай при контрапример" се използва навсякъде (проверка за просто число, дали масив е сортиран и т.н.). Забележка: тук не слагаме break след onlySix = 0;, защото вътрешният цикъл трябва да прочете с scanf и да сумира всичките m оценки — ако прекъснем по-рано, останалите оценки на ученика ще останат непрочетени и входът ще се разчете грешно.

10Структури

Дефиниция

Групира разнородни полета под едно име:

typedef struct {
    int number;
    char name[30];
    float average;
} Student;

Достъп до полета — операторът .

Student s1;
s1.number = 1;
strcpy(s1.name, "Ivan");

Масив от структури

Списък от обекти (напр. цял клас ученици):

Student class_[30];
class_[i].average = sum / m;   // всеки елемент е цяла структура

Указател към структура — операторът ->

void print_student(Student *p) {
    printf("%s: %.2f", p->name, p->average);
}
Кратко сравнение: масив = еднакви елементи (a[i]); структура = един обект с различни полета (s.field); масив от структури = списък от обекти (arr[i].field).

Пример: оценки на клас със структури

Същата задача като при масивите (средна оценка за класа + брой ученици само с 6-ци), но всеки ученик е обект Student с полета name, grades, average и onlySix. Данните за един ученик „пътуват заедно".

Всеки елемент от масива class_[i] е цяла структура; неговите полета се достъпват с точка: class_[i].average, class_[i].grades[k].

#define N 30
#define M 5

typedef struct {
    char  name[30];
    float grades[M];
    float average;      // среден успех на ученика
    int   onlySix;      // 1 = само 6-ци, 0 = не
} Student;

int main() {
    Student class_[N];
    int n, i, k;
    float classSum = 0;   // сума на средните успехи
    int allSixes = 0;      // брой отличници

    scanf("%d", &n);       // брой ученици

    for (i = 0; i < n; i++) {           // външен цикъл - по ученици
        scanf("%s", class_[i].name);
        float sum = 0;
        class_[i].onlySix = 1;         // флаг: приемаме "да"

        for (k = 0; k < M; k++) {       // вътрешен цикъл - по предмети
            scanf("%f", &class_[i].grades[k]);
            sum += class_[i].grades[k];
            if (class_[i].grades[k] != 6)
                class_[i].onlySix = 0;   // оценка != 6 -> не е отличник
        }

        class_[i].average = sum / M;      // успех на ученика
        classSum += class_[i].average;    // трупаме към класа
        if (class_[i].onlySix)
            allSixes++;
    }

    float classAvg = classSum / n;        // среден успех на класа

    printf("Среден успех на класа: %.2f\n", classAvg);
    printf("Отличници (само 6-ци): %d\n", allSixes);
    return 0;
}
Среден успех на класа: 5.24
Отличници (само 6-ци): 3
Разлика с версията само с масиви: тук average и onlySix се пазят вътре в структурата на всеки ученик — остават достъпни и след цикъла (напр. за отпечатване или сортиране). Средният успех на класа смятаме като средно от успехите на учениците (classSum / n); при равен брой предмети M това е същото като „сума на всички оценки / (n·M)".

Блок-схема 1 — обхождане на класа (външен цикъл)

Общият поток: за всеки ученик извикваме „обработка" (детайлите са в блок-схема 2), после трупаме към класа и броим отличниците.

Блок-схема 2 — обработка на един ученик (вътрешен цикъл)

Тук е логиката вътре в един ученик: четем оценките му, трупаме сумата и с флага onlySix проверяваме дали всичките са 6.
