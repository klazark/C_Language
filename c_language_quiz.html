Тема 1: Структура на C програма (Program Structure)
1
Коя е правилната структура на минимална C програма?
Which is the correct structure of a minimal C program?
/* Опции за отговор: */ A) #include <stdio.h> int main() { return 0; } B) main() { printf("Hello"); } C) #include <stdio.h> void main() { return 0; } D) int main { return 0; }
Верен отговор: A

Защо A е правилно: Всяка стандартна C програма трябва да има:

  • #include <stdio.h> (preprocessor directive) - включва библиотеката за вход/изход
  • int main(void) или int main() - главната функция връща цяло число
  • return 0; - показва успешно изпълнение на програмата

Защо другите са грешни:

  • B: Липсва #include за printf и main трябва да връща int според стандарта C99 и по-нови
  • C: void main() не е стандартно - main трябва да връща int, а return 0 в void функция е грешка
  • D: main е функция и изисква скоби main() дори без параметри

Кои от следните декларации на променливи са правилни в C?
Which variable declarations are correct in C?
/* Кои редове са валидни? */ 1) int broj = 10; 2) float cena = 5.99; 3) char bukva = "A"; 4) int 2broj = 5; 5) float sredno = 0.0f;
Верен отговор: A (1, 2 и 5)

Анализ на всеки ред:

  • 1) int broj = 10; ✓ Правилно - целочислена int (integer) променлива
  • 2) float cena = 5.99; ✓ Правилно - float съхранява дробни числа
  • 3) char bukva = "A"; ✗ Грешно! char използва единични кавички 'A'. Двойни кавички "A" са за низове (strings)
  • 4) int 2broj = 5; ✗ Грешно! Името на променлива не може да започва с цифра
  • 5) float sredno = 0.0f; ✓ Правилно - суфиксът f показва, че е float литерал

Тема 3: Вход и Изход (Input/Output Operations)

Какво ще изведе следният код?
What will this code output?
#include <stdio.h> int main(void) { int a = 5, b = 3; float rezultat = (float)a / b; printf("Rezultat: %.2f\n", rezultat); printf("Sumata e: %d\n", a + b); return 0; }
Верен отговор: A

Анализ на кода:

  • (float)a / b - type casting (преобразуване на тип) на a към float прави цялото деление дробно: 5.0 / 3 = 1.666...
  • %.2f - форматира float с 2 цифри след десетичната точка → 1.67
  • %d - целочислен форматен спецификатор за printf
  • a + b = 5 + 3 = 8

Защо другите са грешни:

  • B: Би било вярно без (float) кастинга - тогава 5/3 = 1 (цялочислено деление)
  • C: %.2f закръгля до 2 знака, не показва всички
  • D: %d за printf извежда стойността на израза, не конкатенира цифри
5
Коя е правилната употреба на scanf() за четене на променливи?
What is the correct scanf() usage?
/* Кои scanf извиквания са правилни? */ int n; float x; char c; A) scanf("%d", n); B) scanf("%f", &x); C) scanf("%c", &c); D) scanf("%d %f", &n, x);
Верен отговор: C (B и C)

Ключово правило за scanf: scanf изисква АДРЕСИ (addresses) на променливите, не техните стойности. Използваме оператора & (address-of operator)!

  • A) scanf("%d", n); ✗ Грешно! Липсва & - scanf получава стойността на n вместо адреса
  • B) scanf("%f", &x); ✓ Правилно! &x е адресът на float променливата x
  • C) scanf("%c", &c); ✓ Правилно! &c е адресът на char променливата c
  • D) scanf("%d %f", &n, x); ✗ Грешно! x трябва да е &x, за да се запише прочетената стойност

Важно: При четене на низове (char[]) & не се използва, защото името на масива вече е адрес!

Тема 4: Оператори (Operators)
6
Каква е стойността на suma след изпълнение на кода?
What is the value of 'suma' after execution?
int a = 5, b = 3; int suma = 0; suma = a++ + ++b;
Верен отговор: B (9)

Анализ на инкремент операторите:

  • a++ (post-increment) - първо използва стойността (5), после увеличава a с 1
  • ++b (pre-increment) - първо увеличава b с 1 (от 3 на 4), после използва стойността (4)

Изчисление:

suma = a++ + ++b; suma = 5 + 4; // a++ дава 5 (старта стойност) // ++b дава 4 (увеличена стойност) suma = 9; // След операцията: a = 6, b = 4

Защо другите са грешни:

  • A: Това би било ако и двата бяха пост-инкремент или без инкремент
  • C: Това би било ако и двата бяха пре-инкремент (++a + ++b)
  • D: Това няма как да се получи с тези стойности

Какъв е резултатът от 5 / 2 и 5.0 / 2 в C?
What is the result of 5/2 and 5.0/2 in C?
Верен отговор: B (2 и 2.5)

Важно правило в C: Ако И двата операнда са цели числа, резултатът е целочислено деление (integer division)!

  • 5 / 2 - и двата операнда са int → целочислено деление → 2 (закръгля се надолу)
  • 5.0 / 2 - 5.0 е double, 2 се преобразува до double → дробно деление → 2.5

Допълнителни примери:

7 / 2 → 3 (цялочислено) 7.0 / 2 → 3.5 (дробно) 7 / 2.0 → 3.5 (дробно) (float)7 / 2 → 3.5 (дробно, с кастване)
Тема 5: Условни оператори (Conditional Statements)
8
Какво ще изведе следният код?
What will this code output?
int x = 10, y = 20; if (x > 5) if (y > 15) printf("A\n"); else printf("B\n"); else printf("C\n");
Верен отговор: A

Анализ на вложени if-else:

  • else винаги се свързва с НАЙ-БЛИЗКИЯ предходен if без свой else (dangling else rule)

Стъпка по стъпка:

x = 10, y = 20; if (x > 5) // 10 > 5 е TRUE, влизаме if (y > 15) // 20 > 15 е TRUE printf("A"); // Изпълнява се това! else // този else принадлежи на if (y > 15) printf("B"); else // този else принадлежи на if (x > 5) printf("C");

Важно: Използвайте винаги къдрави скоби {} за яснота при вложени if-else структури!

Какво прави break в switch конструкция?
What does break do in a switch statement?
switch(day) { case 1: printf("Понеделник\n"); case 2: printf("Вторник\n"); break; case 3: printf("Сряда\n"); break; }
Верен отговор: B

switch и break:

  • Без break се получава "проваляне" (fall-through) - изпълняват се всички следващи case-ове!

Пример: Ако day = 1:

// Без break в case 1: Понеделник ← от case 1 Вторник ← проваляне от case 2 (fall-through!) // С break (правилно): Понеделник ← само case 1 се изпълнява

Защо другите са грешни:

  • A: break не спира програмата - това прави exit() или return от main
  • C: break излиза от switch, не прескача case
  • D: break има критично важен ефект - спира fall-through

Каква е разликата между while и do-while?
What is the difference between while and do-while?
/* Вариант A */ int i = 10; while (i < 5) { printf("A"); i++; } /* Вариант B */ int i = 10; do { printf("B"); i++; } while (i < 5);
Верен отговор: B

Разлика между while и do-while:

Цикъл Проверка на условието Минимални итерации
while Преди тялото (pre-test) 0
do-while След тялото (post-test) 1

Анализ на примерите:

Тема 7: Масиви (Arrays)
12
Какво ще изведе следният код?
What will this code output?
int arr[5] = {10, 20, 30, 40, 50}; printf("%d\n", arr[1]); printf("%d\n", arr[5]);
Верен отговор: C

Индексиране на масиви в C:

  • Индексирането в C започва от 0 (zero-based indexing)!
Индекс: 0 1 2 3 4 Стойност: [10] [20] [30] [40] [50] arr[0] = 10 (първи елемент) arr[1] = 20 (втори елемент) arr[4] = 50 (пети/последен елемент) arr[5] = ??? (ИЗВЪН ГРАНИЦИТЕ!)

Анализ:

  • arr[1] → 20 (вторият елемент)
  • arr[5] → Undefined Behavior (undefined behavior)! Масивът има индекси 0-4, 5 е извън границите

Важно: C не проверява граници на масиви! Достъпът извън границите може да:

  • Прочете боклук от паметта
  • Доведе до срив на програмата (segmentation fault)
  • Повреди други данни (security проблем)

Тема 8: Функции (Functions)
14
Какво ще се случи при изпълнение на този код?
What will happen when this code runs?
void swap(int a, int b) { int temp = a; a = b; b = temp; } int main(void) { int x = 5, y = 10; swap(x, y); printf("x=%d, y=%d\n", x, y); return 0; }
Верен отговор: B (x=5, y=10)

Предаване по стойност (Pass by Value):

  • В C аргументите се предават по стойност (pass by value) - функцията получава КОПИЯ на стойностите
  • Промените върху параметрите НЕ засягат оригиналните променливи!

Анализ:

main: swap: x=5 ----┐ a=5 (копие) y=10 ---┘ b=10 (копие) temp = a = 5 a = b = 10 // променя се КОПИЕТО a b = temp = 5 // променя се КОПИЕТО b // След return: x и y в main са НЕПРОМЕНЕНИ!

Правилно решение за размяна: Използвайте указатели (pointers):

void swap_correct(int *a, int *b) { int temp = *a; *a = *b; *b = temp; } // Извикване: swap_correct(&x, &y);

Какво е предназначението на функционен прототип (function prototype)?
What is the purpose of a function prototype?
float stepen(float a, int n); /* това е прототип */ int main() { float r = stepen(2.0, 3); /* извикване преди дефиницията */ return 0; } float stepen(float a, int n) { /* реална дефиниция */ /* ... */ return rezultat; }
Верен отговор: B

Функционен прототип (Function Prototype):

  • Декларира интерфейса на функцията преди нейната дефиниция
  • Позволява извикване на функцията преди нейната реализация в кода
  • Помага на компилатора да провери типовете на аргументите

Синтаксис:

връщан_тип име_функция(тип_парам1, тип_парам2, ...); // Примери: float stepen(float a, int n); // приема float и int, връща float void print_hello(void); // без параметри, не връща нищо int max(int a, int b); // приема два int, връща int

Защо другите са грешни:

  • A: Прототипите не влияят на скоростта на изпълнение
  • C: Прототипът е само декларация, не замества дефиницията
  • D: Прототипът не създава функция, само я декларира
Тема 9: Указатели (Pointers)
16
Какво прави следният код?
What does the following code do?
int a = 10; int *p = &a; *p = 20; printf("%d\n", a);
Верен отговор: B (20)

Анализ на указателите:

int a = 10; // a съхранява стойност 10 int *p = &a; // p съхранява АДРЕСА на a (&a) // p "сочи към" a (p "points to" a) *p = 20; // *p = "стойността на адреса, към който сочи p" // т.е. променяме стойността на a през указателя! printf("%d", a); // a вече е 20

Оператори при работа с указатели:

  • & (address-of operator) - взима адреса на променлива
  • * (dereference/indirection operator) - достъпва стойността на адреса

Визуализация:

Памет: ┌─────────┬──────────┬─────────┐ │ Адрес │ Име │ Стойност│ ├─────────┼──────────┼─────────┤ │ 0x1000 │ a │ 20 │ ← *p модифицира тук │ 0x2000 │ p │ 0x1000 │ ← p съхранява адреса на a └─────────┴──────────┴─────────┘
17
Как да предадем променлива по референция (reference) в C?
How to pass a variable by reference in C?
Верен отговор: C

Предаване по референция в C (чрез указатели):

// Функция приема УКАЗАТЕЛ (адрес) void func(int *x) { // *x означава "указател към int" *x = 10; // *x = достъп до стойността на адреса } // Извикване с адрес на променливата int a = 5; func(&a); // &a = адресът на a // След func: a = 10

Защо другите са грешни:

  • A: Това е предаване по стойност - x е копие, a не се променя
  • B: int &x е синтаксис на C++, не на C! C няма референции като C++
  • D: &x = 10 е невалиден синтаксис - & връща адрес, не може да се присвоява

Важно: C няма вградено "pass by reference" като C++. В C използваме указатели за тази цел!

Тема 11: Обхват на променливи (Variable Scope)
20
Какво ще изведе следният код?
What will this code output?
#include <stdio.h> int x = 10; // глобална променлива void func() { int x = 20; // локална променлива printf("%d ", x); } int main(void) { printf("%d ", x); func(); printf("%d", x); return 0; }
Верен отговор: B (10 20 10)

Обхват на променливи (Variable Scope):

  • Глобални променливи (global) - видими навсякъде във файла
  • Локални променливи (local) - видими само в блока/функцията където са дефинирани
  • Локалните скриват (shadow) глобалните със същото име!
int x = 10; // глобална x = 10 void func() { int x = 20; // НОВА локална x = 20 (скрива глобалната) printf("%d ", x); // → 20 (локалната) } int main(void) { printf("%d ", x); // → 10 (глобалната) func(); // → 20 (локалната в func) printf("%d", x); // → 10 (глобалната, не е променена!) }

Обхват на видимост:

  • main вижда глобалната x (10)
  • func вижда собствената си локална x (20) и игнорира глобалната
  • Промяната в func не засяга глобалната x
